# Emergency Flow — Deploy Guide

## Prerequisites
- GitHub repository with the project
- Vercel account (vercel.com)
- Railway account (railway.app)

## Step 1 — Railway setup (backend + PostgreSQL)

1. Create new project on Railway
2. Add PostgreSQL plugin — Railway generates `DATABASE_URL` automatically
3. Create a new service pointing to this GitHub repo
4. Set root directory to `/backend`
5. Add environment variables:
```
   DATABASE_URL=<auto-generated by Railway PostgreSQL>
   FRONTEND_URL=https://your-app.vercel.app
   NODE_ENV=production
```
6. Railway will build using the Dockerfile and run migrations automatically

## Step 2 — Vercel setup (frontend)

1. Import GitHub repo on Vercel
2. Framework: Next.js (auto-detected)
3. Add environment variable:
```
   NEXT_PUBLIC_API_URL=https://your-backend.railway.app
```
4. Deploy

## Step 3 — GitHub Actions secrets

Add these secrets to your GitHub repo (Settings → Secrets → Actions):
```
VERCEL_TOKEN        → Vercel dashboard → Settings → Tokens
VERCEL_ORG_ID       → Vercel dashboard → Settings → General
VERCEL_PROJECT_ID   → Vercel project → Settings → General
RAILWAY_TOKEN       → Railway dashboard → Account → Tokens
```

## Step 4 — Trigger first deploy
```bash
git add .
git commit -m "feat: production deploy configuration"
git push origin main
```

GitHub Actions will:
1. Run 84 tests
2. TypeScript check frontend + backend
3. Deploy frontend to Vercel
4. Deploy backend to Railway

## Verify deploy
```bash
# Health check
curl https://your-backend.railway.app/api/health

# Test decision
curl -X POST https://your-backend.railway.app/api/decision \
  -H "Content-Type: application/json" \
  -d '{"emergencyType":"heart_attack"}'
```
